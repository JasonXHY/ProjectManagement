# 项目分析与设计文档 —— 轻量级 AI 项目管理工具（Windows 版）

> 本文档基于用户与 DeepSeek 的完整对话生成，涵盖项目需求、技术选型、架构设计、风险评估、可行性验证及开发实施路径。适用于 Claude Code 作为项目启动的基准上下文。

## 一、项目背景与目标

### 1.1 用户原始需求（摘要）

- 在 Windows 上运行一个**轻量级、流畅**的项目管理软件
- 核心功能：
  - 项目页面显示**甘特图**和项目进行情况
  - 支持对接 **Token Plan 或云端大模型 API**（用户自行购买）
  - **拖拽文件**到对应项目，调用云端模型自动**分类文件**
  - **每个项目拥有独立对话框**，可通过自然语言与模型交互（例如“帮我找一下最新的项目蓝图”）
  - 具备**数据分析能力**（通过云端模型）
  - 每个项目采用类似 **Agent 架构的文件夹**，内部存储 **Markdown 文件**记录关键信息
- 设计原则：
  - 轻量级、办公本流畅运行
  - 尽量使用免费/开源组件，仅云端大模型需要用户付费
  - 尽可能节省 API Token 消耗

### 1.2 适用人群与场景

- **目标用户**：项目经理、实施顾问（主要面向信息化项目）
- **两种使用场景**：
  1. **存量项目导入**：用户提供已有项目文档的文件夹路径，程序自动读取、分析、分类，生成项目计划图等
  2. **新项目实时管理**：新建项目后，通过拖拽文件 + 对话的方式逐步构建项目知识库

### 1.3 扩展功能设想（用户补充）

- 页面需要支持查看**项目待处理问题、问题清单、技术方案**等
- 大部分内容可用 Markdown 文件存储，但需要设计专门的页面和按钮来展示
- 预置部分 **md 文件作为上下文关键词**，保证 AI 输出的专业性和一致性

## 二、整体设计思路

### 2.1 核心原则

- **本地优先 + 按需调用大模型**：所有基础数据（项目元信息、md 文档索引）存本地，只在真正需要 AI 理解或生成时才调用云端 API。
- **Token 极致优化**：通过 RAG、分层记忆、Prompt 精简等策略，将每次调用 token 控制在 300～1500 之间，日均成本低于 0.1 元。
- **轻量级运行**：内存占用控制在 30～50 MB，启动 <0.5 秒。

### 2.2 技术选型

| 类型               | 推荐技术                                     | 理由                                                         |
| ------------------ | -------------------------------------------- | ------------------------------------------------------------ |
| **桌面应用框架**   | Tauri 2.0 + React + TypeScript               | 安装包 3-5 MB，内存 30-50 MB，启动 <0.5s                     |
| **UI 组件库**      | Ant Design X（对话组件） + shadcn/ui         | 专业对话组件 + 轻量美观的中后台组件                          |
| **甘特图库**       | `ts-gantt-engine` 或基于 Canvas 的轻量方案   | 支持 300+ 任务节点保持 60fps，体积极小                      |
| **本地数据库**     | SQLite（通过 `rusqlite` 或 `better-sqlite3`） | 无服务，单文件，极轻量                                      |
| **向量数据库**     | LEANN / Zvec / Qdrant Edge                   | 嵌入式，无需独立服务，支持在笔记本上高效检索               |
| **大模型 API**     | OpenAI 兼容协议（如 GLM-4.7-Flash、MiniMax M3） | 用户自行购买 Token，支持免费额度测试                        |
| **文档预处理**     | `pdf-extract`, `textract`                    | 本地提取文本概要，不消耗 Token                              |
| **RAG 切分与检索** | 自研分段策略 + 向量检索                       | 按段落/标题切分，仅召回 Top‑K 片段                          |

## 四、核心模块详细设计

### 4.1 项目页面：甘特图 + 进度展示

- **甘特图设计**：使用轻量级 Canvas 渲染库（如 `ts-gantt-engine`），支持任务依赖、进度百分比、拖拽调整日期。
- **数据结构**（SQLite）：
  ```sql
  projects (id, name, description, created_at, updated_at)
  tasks (id, project_id, name, start_date, end_date, progress, dependency_ids, status)
  milestones (id, project_id, name, date, is_completed)
进度跟踪：手动或通过对话 AI 辅助更新任务进度。

4.2 文件分类系统（拖拽 + 云端分类）
流程：

前端 Drag & Drop 捕获文件路径 → 传递给 Rust 后端

后端批量处理（攒够 5 个或 3 分钟）→ 提取文件概要（本地，不调用 API）

调用大模型 API 进行极简分类（使用预设 Prompt，仅输出 2-4 个字的分类标签）

根据标签将文件复制到项目目录下的对应子文件夹（如 docs/, designs/, meeting_notes/ 等）

同时将文件内容切分后存入本地向量数据库，供后续 RAG 检索

分类 Prompt 模板（节省 Token）：

text
你是文档分类助手。根据以下文件信息和项目上下文，输出一个最合适的分类标签，只输出标签名称，用中文，不超过4个字。
项目名称：[项目名]
文件类型：[PDF/Word/Code/Txt]
文件名：[文件名]
文件开头内容摘要：[最多300字符]
4.3 项目专属对话窗（核心 AI 能力）
每个项目拥有独立的对话会话，历史记录隔离。

对话处理流程：

用户输入问题 → 前端发送至后端
后端执行向量检索：将问题向量化，从该项目的本地向量库中召回 Top-K（K=3~5）相关内容片段
组装 Prompt：系统预置角色指令 + 召回片段 + 项目元信息 + 最近对话历史
调用大模型 API → 流式返回给前端
用户可选择将 AI 回答保存为 Markdown 文件存入项目目录
角色与约束（预置在系统 Prompt 中）：

text
你是一个专业的项目管理助手，专门协助项目经理和实施顾问管理信息化项目。
回答应简洁、专业、基于提供的项目上下文。如果信息不足，请明确说明。
使用 markdown 格式输出，必要时可列出待办事项或问题清单。
4.4 Token 优化方案（核心省钱设计）
策略	实现方式	Token 节省效果
RAG 检索	仅召回 Top‑K 相关片段（数百字），不发送全量文档	减少 90%+
本地结构化记忆	倒排索引、BM25 轻量检索，部分查询不调用模型	零 Token
分层记忆注入	L0~L4 分层，仅注入必要层次	额外 token < 330
Prompt 精简	固定 System Prompt，动态 Context 压缩，代码/文档压缩	额外降低 30%
成本预估（按主流模型 0.002元/1K Tokens）：

每次对话平均消耗 300～1500 Tokens → 0.0006～0.003 元/次

日均 50 次调用 → 0.03～0.15 元/天 → 每月不足 5 元

五、性能与可行性评估
5.1 性能指标（目标值）
指标	目标值
安装包体积	< 10 MB
冷启动内存	30-50 MB
空闲 CPU	< 1%
甘特图渲染（300任务）	60 fps 无卡顿
向量检索（1000文档）	< 200 ms
对话响应（调用API）	取决于网络 + 模型延迟
5.2 风险分析与缓解
风险	影响	缓解措施
大量文档（>5000份）首次索引	CPU 短暂跑满 30 秒，可能影响体验	后台静默索引 + 按需索引；分批次处理，不阻塞 UI
超大 PDF（>500MB）解析	内存占用飙升，解析缓慢	限制提取前 1000 字符，超过 50MB 的文件跳过深层解析
API 调用延迟高或超时	对话窗口等待时间过长	增加超时重试机制；支持离线模式（仅本地检索不调用模型）
本地向量库损坏	检索失效，部分功能不可用	定期备份向量索引；提供重建索引按钮
Windows 环境兼容性	部分老系统缺少 WebView2（需安装 Edge）	安装包检测并提供下载链接；或降级使用 WebView2 旧版兼容
