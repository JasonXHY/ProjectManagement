# G7 — 文件分类阶段排序 UI

> 关联需求：F14.1、F04.2（§4.2 第4条："新增、删除、排序"）
> 依赖：无（独立）
> 阻塞：无
> 优先级：P3

## 背景

F04.2/F14.1 要求文件分类阶段支持"新增、删除、排序"。当前 `SettingsPage.tsx:528` 只有增/删，无排序。

## Scope

### 包含
1. `src/components/Settings/SettingsPage.tsx` 文件分类管理 tab：为阶段列表增加排序能力（上下移按钮，或拖拽）。
2. 排序结果持久化到 `custom_stages`（已在白名单），保持顺序。
3. 顺序应用于新建项目的目录创建顺序与侧边栏展示顺序。

### 不包含
- 子分类排序（本任务只阶段排序）。
- 引入重型拖拽库（优先用上下箭头实现；如用 dnd-kit 需评估包体积）。

## 成功标准
- [ ] 阶段可上移/下移（或拖拽）改变顺序
- [ ] 顺序持久化，重启后保留
- [ ] 默认阶段仍不可删除（不回归 F14.1 保护逻辑）
- [ ] 组件测试：移动后顺序变化并触发保存
- [ ] `npm run typecheck && npm run lint && npm test` 全绿

```bash
npm run typecheck && npm run lint && npm test
```
