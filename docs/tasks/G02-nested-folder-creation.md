# G2 — 嵌套子分类文件夹创建

> 关联需求：F01.1、F04.2（§4.2 文件夹结构）
> 依赖：G1
> 阻塞：G10
> 优先级：P1

## 背景

v3.1 §4.2 规定项目文件夹必须是**两级嵌套**结构：`项目/阶段/子分类/`（见 spec 的文件夹树）。当前 `project-handlers.ts:55` 只建单级扁平阶段目录。

## Scope

### 包含
1. `electron/ipc/project-handlers.ts` 的 `project:create`：按 G1 的 `StageDef[]`（或用户自定义）创建 `阶段/子分类/` 两级目录。
2. 阶段分类方式（`category_type === 'stage'`）时生效；按内容分类仍由 AI 动态创建（保持现状）。
3. 目录名经 `sanitizeFileName` 处理（防非法字符）。
4. 失败时维持现有 DB 回滚逻辑（`project-handlers.ts:73`）。
5. 若用户已自定义子分类（G4 写入 settings），创建时读取自定义结构；否则用默认。

### 不包含
- 已有项目的目录迁移（明确不做，"已有项目不受影响"）。
- 文件移动到子分类目录的逻辑（属于分类流程，G3/G6 涉及 stored_path，本任务只建空目录树）。

## 成功标准
- [ ] 新建「按阶段分类」项目后，磁盘出现完整两级结构（如 `售前/报价单/`、`验收/验收材料待签/`）
- [ ] `.ai/` 及其子目录仍正确创建
- [ ] 文件系统创建失败时数据库回滚，无孤儿记录
- [ ] 用户自定义子分类时按自定义结构创建
- [ ] G10 中有 E2E 用例断言目录树（阶段数、子分类目录存在）
