# PMAer v0.2.0 版本规划

> 创建时间：2026-06-22 | 状态：方案草案（待Qoder审核）
> 前置版本：v0.1.2（首版发版） | 目标：业务卡片增强 + 项目转交 + 细节优化

---

## 一、版本定位

v0.2.0 是 PMAer 的**业务深化版本**，核心目标：
1. **项目卡片信息增强** — 让项目经理一眼掌握项目全貌
2. **项目转交功能** — 支持团队协作，新成员快速了解项目
3. **Markdown预览优化** — 提升文档查看体验
4. **细节打磨** — 修复遗留问题，提升整体质量

## 二、功能模块详细设计

### 模块A：项目卡片信息增强

#### A1. 卡片信息层次设计

**当前卡片信息**：项目名称 + 状态标签 + 分类方式 + 更新时间 + 甘特图

**新增信息维度**（按优先级）：

| 层级 | 信息 | 数据来源 | 展示方式 |
|------|------|----------|----------|
| L1-核心 | 项目名称 | projects.name | 标题 |
| L1-核心 | 项目状态 | projects.current_stage | 彩色标签 |
| L1-核心 | 关键里程碑 | projects.milestones (JSON) | 进度条/时间轴 |
| L2-概览 | 文件总数 | files COUNT | 数字 |
| L2-概览 | 已分类文件数 | files WHERE category IS NOT NULL | 数字 + 百分比 |
| L2-概览 | 待处理文件数 | files WHERE category IS NULL | 数字（红色高亮） |
| L2-概览 | AI分析状态 | files WHERE is_analyzed = 0 | 进度提示 |
| L3-详情 | 最近活动 | files MAX(created_at) | 时间 |
| L3-详情 | 关键问题数 | .ai/project-summary.md 解析 | 数字（如有） |
| L3-详情 | 签字进度 | files WHERE signature_status = 'signed' / total | 分数 |
| L3-详情 | 项目创建时间 | projects.created_at | 时间 |

#### A2. 卡片布局方案

```
┌─────────────────────────────────────────────┐
│ ████ 项目名称                          状态 │  ← 彩色条 + 标题 + 状态标签
│                                             │
│ 📁 12 文件 | ✅ 8 已分类 | ⚠️ 4 待处理     │  ← 文件统计行
│                                             │
│ ━━━━━━━━━━━━━░░░░ 75%                      │  ← 里程碑进度条
│ 需求确认 → 方案设计 → 构建测试              │
│                                             │
│ ✍️ 签字 2/5 | 🔍 待分析 3 | 📅 3天前更新   │  ← 详情行
│                                             │
├─────────────────────────────────────────────┤
│ 3天前更新                    [管理] [编辑]   │  ← 底部操作栏
└─────────────────────────────────────────────┘
```

#### A3. 里程碑进度条

从 `projects.milestones` JSON 解析，计算完成百分比：

```typescript
interface Milestone {
  title: string
  date: string        // YYYY-MM-DD
  completed: boolean  // 新增字段
}

// 进度 = completed数 / total数 * 100%
```

**展示规则**：
- 无里程碑数据：不显示进度条
- 有数据：显示进度百分比 + 关键节点时间轴
- 节点超过3个时只显示首尾 + 当前阶段

#### A4. 待分类文件高亮

- 当有待处理文件时，卡片边框变为橙色
- 点击卡片可直接进入项目查看待处理文件
- 数字角标显示在卡片右上角

#### A5. 签字进度追踪

- 分数显示：`已签 N / 总数 M`
- 全部已签：绿色
- 部分已签：橙色
- 无签字文件：灰色

### 模块B：项目转交功能（核心新功能）

#### B1. 功能概述

**场景1：整体转交** — 项目经理将整个项目交给接手人
- 打包所有项目文件 + 元数据 + AI摘要
- 生成标准格式的转交包
- 接手人导入后可完整还原项目上下文

**场景2：协作转交** — 邀请同事参与项目特定部分
- AI根据描述智能选择相关文件
- 生成精简的转交包（仅包含必要信息）
- 接手人快速了解项目背景和负责模块

#### B2. 转交包格式设计

**格式选择**：JSON + 文件附件（非压缩包）

**理由**：
1. JSON是AI易读格式，云端模型可直接解析
2. 结构化数据便于导入和校验
3. 文件保持原始格式，无需解压

**转交包结构**：

```
pmaer-handover/
├── handover.json          ← 转交元数据（必须）
├── project-summary.md     ← 项目摘要（AI生成）
├── key-info.md           ← 关键信息（客户/合同/联系人）
├── files/                 ← 项目文件（按分类组织）
│   ├── 售前/
│   │   ├── 报价单.xlsx
│   │   └── 需求文档.docx
│   ├── 需求/
│   │   └── 需求规格说明书.docx
│   └── ...
└── context.md            ← 转交说明（发送方填写的描述）
```

**handover.json 结构**：

```json
{
  "format_version": "1.0",
  "type": "full" | "selective",
  "project": {
    "name": "项目名称",
    "category_type": "stage",
    "current_stage": "进行中",
    "created_at": "2026-06-01T10:00:00Z",
    "milestones": [...],
    "metadata": {...}
  },
  "files": [
    {
      "filename": "需求文档.docx",
      "category": "需求",
      "subcategory": "需求规格",
      "stage": "需求",
      "file_type": "application/docx",
      "file_size": 102400,
      "content_extracted": "（提取的文本内容）",
      "signature_status": "signed",
      "ai_summary": "（AI摘要）",
      "ai_key_info": "（关键信息）"
    }
  ],
  "summary": {
    "project_overview": "（AI生成的项目概览）",
    "key_clients": "（关键客户信息）",
    "challenges": "（项目难点）",
    "pending_items": "（待办事项）",
    "team_context": "（团队上下文）"
  },
  "handover_note": "（发送方填写的转交说明）",
  "exported_at": "2026-06-22T15:00:00Z",
  "exported_by": "PMAer v0.2.0"
}
```

#### B3. 转交流程设计

**整体转交互动流程**：

```

    ↓

    ↓
┌─ 整体转交 ─┐    ┌─ 协作转交 ─┐
│ 直接打包    │    │ 填写描述    │
│ 所有文件    │    │ ↓          │
│            │    │ AI分析选择  │
│            │    │ ↓          │
│            │    │ 预览确认    │
└────┬───────┘    └────┬───────┘
     ↓                 ↓
生成转交包（文件夹）
     ↓
提供转交链接/文件路径
```

**协作转交的AI决策流程**：

```

    ↓

    ↓

    ↓
返回：需要打包的文件列表 + 理由
    ↓
用户确认/调整
    ↓
生成精简转交包
```

### 模块C：Markdown预览优化

#### C1. 问题现状

当前AI回复已支持Markdown渲染（react-markdown），但：
1. 项目中的.md文件打开仍是纯文本
2. 无语法高亮
3. 无表格渲染
4. 无图片嵌入支持

#### C2. 优化方案

**方案：统一Markdown渲染组件**

```typescript
// 新增组件：src/components/MarkdownPreview/
// - MarkdownPreview.tsx — 主组件
// - code-highlight.ts — 代码高亮配置

<MarkdownPreview 
  content={fileContent}
  fileName="readme.md"
  maxHeight={600}
/>
```

**技术选型**：
- 复用现有 `react-markdown` + `remark-gfm`
- 新增 `rehype-highlight` 用于代码语法高亮
- 新增 `rehype-raw` 支持HTML嵌入

**应用场��**：
1. AI回复渲染（已有，优化样式）
2. .md文件预览（新增）
3. 项目摘要预览（新增）
4. 转交包中的context.md预览（新增）

#### C3. 文件预览入口

在文件列表中，.md文件增加"预览"按钮：

```
📄 README.md    [预览] [打开] [分类]
```

点击"预览"弹出模态框显示渲染后的Markdown。

## 三、项目经理视角：卡片信息价值

### 项目经理关注什么？

作为项目经理，打开PMAer首页时，需要一眼回答以下问题：

| 问题 | 对应卡片信息 | 紧迫度 |
|------|-------------|--------|
| 项目整体进度如何？ | 里程碑进度条 | 高 |
| 有哪些文件待处理？ | 待分类文件数 | 高 |
| 签字流程走到哪了？ | 签字进度 | 高 |
| 最近有什么新文件？ | 最近活动时间 | 中 |
| AI分析有什么发现？ | 关键问题数 | 中 |
| 团队最近在做什么？ | 文件分类分布 | 低 |

### 卡片信息优先级矩阵

```
          高紧迫度
              │
    ┌─────────┼─────────┐
    │ 里程碑  │ 待分类  │
    │ 进度    │ 文件数  │
    │         │         │
高──┼─────────┼─────────┤──低
价  │ 签字    │ 关键    │ 价
值  │ 进度    │ 问题数  │ 值
    │         │         │
    └─────────┼─────────┘
              │
          低紧迫度
```

## 四、顾问视角：转交功能价值

### 新顾问加入项目时需要了解什么？

| 信息类别 | 具体内容 | 转交包提供方式 |
|----------|----------|---------------|
| **项目背景** | 项目目标、范围、时间线 | project-summary.md |
| **客户信息** | 关键联系人、决策者、偏好 | key-info.md + metadata |
| **团队情况** | 分工、接口人、协作方式 | context.md（手动填写） |
| **技术栈** | 使用的技术、架构、规范 | 文件内容提取 |
| **当前状态** | 进行中的工作、待办事项 | AI分析摘要 |
| **历史决策** | 为什么这样做、做过什么选择 | 文件内容 + AI总结 |
| **风险点** | 已知问题、潜在风险 | AI分析 + 手动补充 |
| **验收标准** | 客户验收要求、质量标准 | 需求文档 + 合同 |

### 协作转交的智能选择逻辑

AI应根据新同事的角色选择相关文件：

| 角色 | 优先选择的文件类型 |
|------|-------------------|
| 前端开发 | UI规范、接口文档、设计稿、前端代码 |
| 后端开发 | 接口文档、数据库设计、后端代码 |
| 测试工程师 | 需求文档、测试用例、缺陷列表 |
| 产品经理 | 需求文档、竞品分析、用户反馈 |
| 项目经理 | 全部文件（整体转交更合适） |
| 客户成功 | 合同、验收标准、客户沟通记录 |

## 六、风险与依赖

### 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI文件选择不准确 | 协作转交给出无关文件 | 允许用户手动调整选择 |
| 转交包体积过大 | 大项目文件过多 | 支持选择性转交、压缩 |
| Markdown渲染性能 | 大文件预览卡顿 | 虚拟滚动、分页加载 |
| 跨版本兼容 | 旧版PMAer无法导入新版转交包 | 版本号校验 + 向后兼容 |

## 九、总结

v0.2.0 是 PMAer 从"个人工具"向"团队协作工具"演进的关键版本。项目转交功能将打破信息孤岛，让项目知识在团队中流动。同时，卡片信息增强让项目经理能够更高效地掌控项目全局。

**核心价值主张**：
- **一眼掌握**：项目卡片让关键信息触手可及
- **知识传递**：转交功能让项目经验可传承
- **智能辅助**：AI帮助筛选最相关的转交内容
