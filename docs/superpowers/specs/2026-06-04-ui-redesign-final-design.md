## 三、文件上传流程

### 3.1 新流程（已确认）

```
用户拖拽文件 → AI自动分类 → 文件保存到对应阶段
```

**无需用户确认**：AI分类后直接保存

### 3.2 手动调整机制

- 用户可以手动调整文件的分类
- 用户手动调整过的分类，以用户为准
- 用户调整过的文件，未来不再参与AI分类

### 3.3 实现要点

1. 修改 `FileManager.tsx`：移除阶段选择要求
2. 修改 `handleUpload`：直接调用AI分类，无需确认
3. 添加 `manual_category` 字段：标记用户手动调整的分类
4. 修改AI分类逻辑：跳过已手动调整的文件

## 四、导航栏设计

### 4.1 设计原则

- **层级清晰**：用户知道当前位置
- **跳转方便**：操作步骤少
- **操作友好**：视觉美观，易于使用
- **响应式**：窗口大小变化时能正常使用

### 4.2 导航结构

#### 项目列表页
```
[项目管理助手] [设置]
```

#### 项目首页
```
[项目管理助手] > [项目名称] [设置]
```
左侧：项目阶段导航栏（支持滚动）

#### 对话页
```
[项目管理助手] > [项目名称] > [对话] [设置]
```

#### 设置页
```
[项目管理助手] > [设置] [返回]
```

### 4.3 响应式设计

- **左侧导航栏**：
  - 固定宽度：120px
  - 内容超出时显示滚动条
  - 窗口高度不足时可上下滚动
  - 不能有遮挡，所有阶段都要能选到

## 五、UI设计原则

### 5.1 视觉设计

- **对比度**：文字和背景要有足够对比度
- **可读性**：字体大小适中，行高合理
- **一致性**：颜色、字体、间距保持一致
- **层次感**：通过颜色、大小、间距体现层级关系

### 5.2 交互设计

- **反馈**：操作后要有明确反馈
- **预测性**：用户能预测操作结果
- **容错**：允许用户撤销操作
- **效率**：减少操作步骤

### 5.3 颜色方案

- **主色**：蓝色系（专业、可信）
- **辅助色**：灰色系（中性、稳重）
- **强调色**：橙色/绿色（操作按钮、成功状态）
- **警告色**：黄色/红色（警告、错误）
