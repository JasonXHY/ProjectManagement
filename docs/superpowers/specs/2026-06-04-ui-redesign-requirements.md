## 二、需求详情

### 2.1 文件上传流程重构

#### 当前流程（有问题）
1. 用户选择项目阶段
2. 用户上传文件
3. AI对文件进行分类
4. 用户确认分类
5. 文件保存到对应阶段

**问题**：用户已经选择了阶段，AI分类就失去意义了。

#### 新流程（已确认）
1. 用户直接上传文件（无需选择阶段）
2. AI自动对文件进行分类
3. 文件直接保存到AI分类的对应阶段
4. **无需用户确认**

#### 手动调整机制
- 用户可以手动调整文件的分类
- 用户手动调整过的分类，以用户为准
- 用户调整过的文件，未来不再参与AI分类

#### 实现要点
- 修改 `FileManager.tsx`：移除阶段选择要求
- 修改 `handleUpload`：直接调用AI分类，无需确认
- 添加 `manualCategory` 字段：标记用户手动调整的分类
- 修改AI分类逻辑：跳过已手动调整的文件

### 2.6 UI设计原则

#### 视觉设计
- 对比度：文字和背景要有足够对比度
- 可读性：字体大小适中，行高合理
- 一致性：颜色、字体、间距保持一致
- 层次感：通过颜色、大小、间距体现层级关系

#### 交互设计
- 反馈：操作后要有明确反馈
- 预测性：用户能预测操作结果
- 容错：允许用户撤销操作
- 效率：减少操作步骤

#### 颜色方案
- 主色：蓝色系（专业、可信）
- 辅助色：灰色系（中性、稳重）
- 强调色：橙色/绿色（操作按钮、成功状态）
- 警告色：黄色/红色（警告、错误）
