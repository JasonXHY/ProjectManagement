# PMAer 卡片系统优化方案

> 创建时间：2026-06-25 | 基于用户测试反馈

---

## 一、问题清单

### 1.1 分类性能问题

**问题描述**：文件数量多时，一键分类速度慢，需要等待较长时间。

**根因分析**：
- 当前实现：串行处理，每个文件单独调用AI分类
- 代码位置：`src/components/ProjectHome/projectHome.hooks.ts` 第159-221行
- 每个文件分类需要：读取文件内容 → 构建prompt → 调用AI → 解析结果 → 更新数据库

**影响范围**：所有需要批量分类的场景

### 1.2 分类准确性问题

**问题描述**：蓝图文件被误分类为"开发规格说明书"。

**根因分析**：
- AI分类prompt中，"方案"阶段的子分类包含"开发规格说明书"和"蓝图"
- AI可能根据文件内容中的技术术语误判
- 代码位置：`electron/prompts/classify.ts` 第24行

**影响范围**：所有蓝图类文件

### 1.3 阶段分类重复问题

**问题描述**：转客户成功和上线阶段都有"操作手册"子分类。

**根因分析**：
- `electron/shared/stages.ts` 第27-29行定义了重复的子分类
- 上线阶段：`['上线切换方案', '部署文档', '操作手册', '签字材料', '初始化材料', '问题清单', '会议纪要']`
- 转客户成功阶段：`['交接文档', '培训资料', 'FAQ']`

**用户需求**：去掉转客户成功的操作手册分类，只保留上线阶段的。

### 1.4 交付物清单统计缺失

**问题描述**：操作手册属于交付文件，但交付物清单没有统计。

**根因分析**：
- DeliverableCard组件只从`project.metadata.deliverables`获取数据
- 没有自动从文件中识别和统计交付物
- 代码位置：`src/components/ProjectHome/cards/DeliverableCard.tsx`

**用户需求**：交付物清单应该自动统计交付给客户的文件（蓝图、操作手册、测试脚本等）。

### 1.5 特色卡片阶段映射问题

**问题描述**：转客户成功和关闭阶段应该共用关闭阶段的特色卡片。

**根因分析**：
- `src/components/ProjectHome/FeatureCards.tsx` 定义了阶段分组
- 当前：转客户成功属于进行中分组，关闭属于关闭分组
- 用户需求：转客户成功和关闭共用关闭阶段的特色卡片

## 二、解决方案

### 2.1 分类性能优化

**方案A：并行分类（推荐）**
- 将串行循环改为并行处理
- 使用`Promise.all`或分批并行
- 预期提升：5-10倍速度提升

**方案B：批量AI调用**
- 将多个文件内容合并到一个prompt中
- 一次性返回多个文件的分类结果
- 预期提升：10-20倍速度提升，但需要修改AI prompt

**方案C：缓存机制**
- 对相似文件名/内容进行缓存
- 避免重复调用AI

**建议**：先实施方案A（并行分类），风险最低，效果明显。

### 2.2 分类准确性优化

**方案A：增强prompt规则（推荐）**
- 在prompt中增加蓝图文件的识别规则
- 明确蓝图 vs 开发规格说明书的区别

**方案B：文件名启发式**
- 在AI分类前，先根据文件名进行预判
- 蓝图文件通常包含"蓝图"、"业务蓝图"等关键词

**方案C：两阶段分类**
- 第一阶段：根据文件名/元数据快速分类
- 第二阶段：AI确认或修正

**建议**：实施方案A+B组合。

### 2.3 阶段分类修正

**修改内容**：
- 移除转客户成功阶段的"操作手册"子分类
- 保留上线阶段的"操作手册"子分类
- 转客户成功阶段子分类：`['交接文档', '培训资料', 'FAQ']`

**涉及文件**：`electron/shared/stages.ts`

### 2.4 交付物清单自动统计

**方案A：基于文件分类自动识别（推荐）**
- 识别规则：蓝图、操作手册、测试脚本、方案文档等
- 文件分类阶段：上线、验收、关闭
- 文件子分类：操作手册、测试报告、方案文档等

**方案B：AI辅助识别**
- 在分类时标记文件是否为交付物
- 新增`is_deliverable`字段

**方案C：手动标记**
- 用户手动标记文件为交付物
- 最灵活但最麻烦

**建议**：实施方案A，同时保留手动标记能力。

### 2.5 特色卡片阶段映射修正（用户确认 2026-06-25）

**修改内容**：
- 为"转客户成功"阶段新增独立的第四组特色卡片（handover 分组）
- 卡片组合：**签字追踪 + 拓展商机 + 交付物清单**
- 业务逻辑：转客户成功阶段的核心关注点是交接与收尾，这三张卡片最匹配

**实现方式**：
- 在 `FeatureCards.tsx` 中新增 `HANDOVER_STAGES = ['转客户成功']` 和 `HANDOVER_CARDS = [SignatureCard, OpportunityCard, DeliverableCard]`
- `StageGroup` 类型扩展为 `'presale' | 'progress' | 'handover' | 'closed'`
- `getStageGroup()` 增加 handover 分支判断
- 从 `PROGRESS_STAGES` 中移除"转客户成功"

**涉及文件**：`src/components/ProjectHome/FeatureCards.tsx`

## 三、实施计划（用户确认 2026-06-25）

### 第一阶段：快速修复（1-2天）

1. **修正特色卡片映射**（`FeatureCards.tsx`）
   - 新增 handover 分组：转客户成功 → 签字追踪 + 拓展商机 + 交付物清单
   - 从 PROGRESS_STAGES 移除"转客户成功"

### 第二阶段：性能优化（2-3天）

2. **实现并行分类**（`projectHome.hooks.ts`）
   - 将串行循环改为并行处理
   - 添加并发控制（避免过多并行导致AI限流）

### 第三阶段：准确性优化（2-3天）

3. **增强分类prompt**（`electron/prompts/classify.ts`）
   - 增加蓝图文件识别规则
   - 明确蓝图 vs 开发规格说明书的区别

4. **添加文件名启发式**
   - 在AI分类前，根据文件名进行预判

### 第四阶段：交付物统计（3-5天）

5. **实现交付物自动识别**
   - 基于文件分类阶段和子分类识别交付物
   - 修改DeliverableCard组件

## 四、设置界面同步（用户确认 2026-06-25）

**结论**：设置页的"文件分类管理"已读取 `STAGE_DEFINITIONS` 作为默认值，`stages.ts` 修改后设置页会自动同步，无需额外开发。

**已知隐患（记录，非本次范围）**：
- `StageSidebar.tsx` 硬编码了阶段列表，未读取设置中的 `custom_stages`。用户在设置中新增/重排阶段后，侧边栏不会更新。建议后续版本修复。
