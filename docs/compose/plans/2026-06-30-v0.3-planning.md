## 三、v0.3.0 功能清单

### 3.1 卡片编辑功能（P0）

**目标**：所有卡片支持点击编辑，用户可修正AI数据

**实现方案**：
- 每个卡片添加"编辑"按钮
- 点击后切换为编辑模式（Input/Select组件）
- 保存时更新metadata，同时更新数据库
- 编辑历史记录（可选，v0.4）

**涉及卡片**（9个）：
1. ProjectInfoCard — 项目名称、客户、负责人、阶段
2. ContractCard — 合同金额、币种、签订日期
3. EvaluationCard — 利润率、成本结构
4. RequirementCard — 需求清单增删改
5. IssueCard — 问题清单增删改+状态切换
6. SignatureCard — 签字状态手动标记
7. OpportunityCard — 商机描述、解决方案
8. DeliverableCard — 交付物状态、版本
9. ProjectSummaryModal — 成果总结、经验教训

### 3.2 AI分类准确率提升（P1）

**目标**：分类准确率从85-90%提升到95%+

**优化方向**：
1. **Prompt优化**：增加few-shot示例，明确边界条件
2. **文件名启发式**：合同/CON→售前，验收/SS→验收，结算→关闭
3. **反馈闭环**：用户手动分类后记录，下次优先匹配
4. **多阶段验证**：分类后检查阶段合理性，不合理时提示用户

### 3.3 内置API Key迁移（P0）

**目标**：7月底前完成迁移，用户无感知切换

**实现方案**：
1. **启动时检测**：如果使用内置Key，弹窗提示"内置Key即将关闭"
2. **一键跳转**：弹窗按钮直接跳转到设置页AI模型tab
3. **Grace Period**：7月31日后内置Key失效，AI功能降级为"未配置"
4. **数据保护**：Key失效不影响已有数据，只影响新分析

### 3.4 测试补全（P1）

**目标**：E2E测试覆盖核心流程

**测试范围**：
1. 项目CRUD + 文件上传 + AI分类（真实IPC + SQLite）
2. 卡片数据读写 + 编辑保存
3. 利润测算完整流程
4. 转交导出/导入端到端
