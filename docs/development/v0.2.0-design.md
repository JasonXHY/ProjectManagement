## 三、模块B：项目转交功能

### B1. 转交包格式：.pmaer.zip

**文件格式**：ZIP压缩包，扩展名 `.pmaer.zip`

**包内结构**：
```
project-name.pmaer.zip/
├── handover.json          ← 转交元数据（必须）
├── project-summary.md     ← AI项目摘要（如有）
├── key-info.md           ← 关键信息汇总（如有）
├── context.md            ← 转交说明（发送方填写）
└── files/                 ← 项目文件（按分类目录组织）
    ├── 售前/
    │   ├── 报价单/
    │   │   └── XX项目报价.xlsx
    │   └── 合同原件/
    │       └── 合同v2.pdf
    ├── 需求/
    │   └── 需求文档/
    │       └── 需求规格说明书.docx
    └── ...
```

### B4. 导出流程

#### 整体转交流程

```
用户点击「转交项目」
    ↓
选择「整体转交」
    ↓
填写转交说明（可选）
    ↓
handoverService.exportFull(projectId)
    ↓
  1. 读取 projects 表获取元数据
  2. 读取 files 表获取所有文件记录
  3. 读取 .ai/project-summary.md（如有）
  4. 生成 key-info.md（从 metadata 提取）
  5. 复制所有文件到临时目录
  6. 生成 handover.json
  7. 打包为 .pmaer.zip
  8. 清理临时目录
    ↓
保存对话框：用户选择保存位置
    ↓
导出完成
```

#### 协作转交流程

```
用户点击「转交项目」
    ↓
选择「协作转交」
    ↓
填写同事描述（必填）
  例："张工负责前端开发，需要了解UI规范和接口文档"
    ↓
填写转交说明（可选）
    ↓
点击「AI推荐文件」
    ↓
handoverService.aiSelectFiles(projectId, description)
    ↓
  1. 读取 files 表（文件名+分类+AI摘要前100字）
  2. 读取 project-summary.md
  3. 构建 prompt（见B5）
  4. 调用 aiService.chat()
  5. 解析 AI 返回的 JSON
  6. 返回推荐文件列表 + 推荐理由
    ↓
渲染文件勾选列表（AI推荐已勾选）
    ↓
用户确认/调整勾选
    ↓
handoverService.exportSelective(projectId, selectedFiles, note)
    ↓
  （与整体转交类似，但只包含 selectedFiles 列表中的文件）
    ↓
保存对话框 → 导出完成
```

### B6. 导入流程

```
用户在首页点击「导入转交」按钮
    ↓
选择 .pmaer.zip 文件（文件选择器或拖拽）
    ↓
handoverService.previewImport(zipPath)
    ↓
  1. 解压 ZIP 到临时目录
  2. 读取 handover.json
  3. 校验 format_version（当前支持 "1.0"）
  4. 返回预览数据
    ↓
┌─────────────────────────────────────┐
│  导入预览                             │
│                                     │
│  项目名称：XX管理系统               │
│  文件数量：23个文件                  │
│  项目阶段：进行中                    │
│                                     │
│  转交说明：                          │
│  "该项目下周需交付，重点关注..."     │
│                                     │
│  [项目摘要预览（前200字）]          │
│                                     │
│  导入后项目名称：[XX管理系统     ]   │
│  （可修改，默认保留原名）            │
│                                     │
│              [取消]  [确认导入]      │
└─────────────────────────────────────┘
    ↓
用户确认导入
    ↓
handoverService.importProject(zipPath, projectName?)
    ↓
  1. 创建新项目（使用用户填写的名称）
  2. 创建项目文件夹
  3. 写入 .uuid 文件
  4. 解压文件到项目文件夹对应阶段/子分类目录
  5. 导入 projects 表元数据
  6. 导入 files 表文件记录（保留分类信息）
  7. 导入里程碑数据
  8. 清理临时目录
    ↓
导入完成，自动进入项目详情页
```

### B7. 冲突处理

| 场景 | 处理方式 |
|------|----------|
| 同名项目已存在 | 提示"项目名称已存在，是否自动重命名为 'XX (导入)'？" |
| format_version不兼容 | 提示"转交包格式版本不兼容，请升级PMAer" |
| ZIP损坏 | 提示"文件格式错误，请确认文件完整性" |
| 文件提取失败 | 跳过该文件，导入其余文件，日志记录 |
