### 2.3 Feature Cards 设计

**由侧边栏选中项驱动，映射到项目阶段分组。**

#### 侧边栏→分组映射逻辑

```typescript
type StageGroup = 'presale' | 'progress' | 'closed'

function getStageGroup(sidebarItem: string, projectStage: string): StageGroup {
  if (sidebarItem === '所有文件') {
    // 跟随项目阶段
    if (projectStage === '售前') return 'presale'
    if (projectStage === '关闭') return 'closed'
    return 'progress'
  }
  if (sidebarItem === '售前') return 'presale'
  if (sidebarItem === '关闭') return 'closed'
  return 'progress' // 启动/需求/方案/构建/测试/上线/验收/转客户成功
}
```

## 四、项目转交功能设计

### 4.1 转交包格式

**文件格式**：ZIP压缩包，扩展名 `.pmaer.zip`

**包内结构**：
```
project-name.pmaer.zip/
├── handover.json          ← 转交元数据（必须）
├── project-summary.md     ← AI项目摘要（如有）
├── key-info.md           ← 关键信息汇总（如有）
├── context.md            ← 转交说明（发送方填写）
└── files/                ← 项目文件（按分类目录组织）
    ├── 售前/
    │   ├── 报价单/
    │   │   └── XX项目报价.xlsx
    │   └── 合同原件/
    │       └── 合同v2.pdf
    ├── 需求/
    │   └── 需求文档/
    │       └── 需求规格说明书.docx
    └── ...
```

### 4.2 handover.json 结构

```typescript
interface HandoverJSON {
  format_version: '1.0'
  type: 'full' | 'selective'
  project: {
    name: string
    category_type: 'stage' | 'content'
    current_stage: string
    created_at: string           // ISO 8601
    milestones: Milestone[]
    metadata: Record<string, any>
  }
  files: HandoverFile[]
  handover_note: string
  exported_at: string            // ISO 8601
  exported_by: string            // "PMAer v0.2.0"
}

interface HandoverFile {
  filename: string
  relative_path: string          // 相对于files/目录
  category: string | null
  subcategory: string | null
  stage: string | null
  file_type: string
  file_size: number
  content_extracted: string | null
  signature_status: string
  ai_summary: string | null
  ai_key_info: Record<string, any> | null
}
```

### 4.5 AI推荐Prompt设计

```
你是PMAer项目转交助手。根据新同事的角色描述，从项目文件中选择最相关的文件。

## 新同事描述
{user_description}

## 项目文件列表（含分类信息）
{file_list_with_categories_and_summaries}

## 项目摘要
{project_summary}

请返回JSON格式：
{
  "selected_files": [
    {
      "filename": "文件名",
      "reason": "选择理由（一句话）"
    }
  ],
  "key_context": "新同事需要了解的关键上下文概述（2-3句话）"
}

选择原则：
1. 优先选择与角色描述直接相关的文件
2. 合同、报价等商务文件仅在明确需要时选择
3. 会议纪要选择最近3次相关的
4. 不超过15个文件
```
