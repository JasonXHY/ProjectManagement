## 一、核心概念说明（重要）

### 1.1 两套阶段体系

本系统存在**两套独立的阶段体系**，用途不同，不可混淆：

#### 项目阶段（Project Stage）— 3个

用于**项目整体状态管理**，表示项目当前所处的宏观阶段：

| 阶段 | 含义 | 触发条件 |
|------|------|----------|
| **售前** | 项目尚未正式立项，处于商机跟踪阶段 | 创建项目时默认 |
| **进行中** | 项目已立项，正在执行中 | AI识别到合同/启动文件时自动推进，或用户手动推进 |
| **关闭** | 项目已完成交付，归档状态 | AI识别到验收/上线文件时自动推进，或用户手动推进 |

**设计原则：**
- 项目阶段保持**粗粒度**（仅3个），因为：
  1. 过细的阶段划分会增加AI判断复杂度
  2. 项目管理本身不需要精确到每个执行步骤
  3. 精细化分类由「文件分类阶段」承担

#### 文件分类阶段（File Classification Stage）— 10个 + 未分类

用于**文件精细化分类**，表示文件内容属于项目生命周期的哪个具体环节。每个阶段下有**子分类**（Sub-category），进一步区分文件类型和状态：

| 阶段 | 子分类 | 区分标准 | 典型文件 |
|------|--------|----------|----------|
| **售前** | 销售方案 | 项目规划、技术路线 | 技术方案、项目规划书 |
| | 报价单 | 费用、报价 | 报价清单、费用明细、人天估算 |
| | 合同原件 | 法律约束文件 | 合同初稿、补充协议、保密协议 |
| | 客户沟通 | 非正式交流 | 邮件往来、沟通纪要、需求意向 |
| | 成本评估 | 内部成本测算 | 成本估算、利润分析、资源评估 |
| | POC材料 | 概念验证 | POC方案、演示材料、试用报告 |
| **启动** | 项目章程 | 立项依据 | 立项审批、项目章程、可行性报告 |
| | 团队组建 | 人员安排 | 团队名单、职责分工、通讯录 |
| | 启动会议 | 启动阶段会议 | 启动会议纪要、议程、签到表 |
| **需求** | 需求文档 | 核心需求定义 | 需求规格、用户故事、用例文档 |
| | 会议纪要 | 需求阶段会议 | 需求评审纪要、需求讨论记录 |
| | 需求变更 | 变更管理 | 变更申请、变更通知、影响分析 |
| | 客户材料 | 客户方提供的输入 | 客户业务材料、客户流程文档、客户数据 |
| | 项目计划 | 进度安排 | 项目计划书、里程碑、WBS、甘特图 |
| **方案** | 开发规格说明书 | 技术实现规格 | 技术选型、架构设计、接口规格 |
| | 蓝图 | 业务+系统设计 | 业务蓝图、系统蓝图、流程设计 |
| | 方案汇报材料 | 向客户/领导汇报 | 方案PPT、汇报材料、演示文档 |
| | 会议纪要 | 方案阶段会议 | 方案评审纪要、设计讨论记录 |
| **构建** | 开发文档 | 开发过程文档 | 代码规范、开发指南、环境说明 |
| | 接口文档 | API定义 | 接口清单、调用说明、数据字典 |
| | 配置文档 | 环境与部署 | 部署配置、环境变量、中间件配置 |
| **测试** | 测试用例 | 测试设计 | 测试方案、用例清单、自动化脚本 |
| | 测试报告 | 测试结果 | 测试报告、缺陷清单、回归记录 |
| | 测试数据 | 测试素材 | 测试数据集、样本文件、mock数据 |
| | 会议纪要 | 测试阶段会议 | 测试评审纪要、缺陷讨论记录 |
| **上线** | 上线切换方案 | 上线策略（含回滚） | 上线方案、切换方案、回滚方案、应急预案 |
| | 部署文档 | 部署执行 | 部署手册、发布说明、环境检查清单 |
| | 操作手册 | 用户操作指南 | 操作手册、使用说明、快速入门 |
| | 签字材料 | 需签字确认 | 上线确认单、变更审批单（待签/已签） |
| | 初始化材料 | 系统初始化 | 数据初始化、参数配置、基础数据导入 |
| | 问题清单 | 上线问题跟踪 | 问题列表、bug清单、处理记录 |
| | 会议纪要 | 上线阶段会议 | 上线评审纪要、上线协调会记录 |
| **验收** | 验收材料（待签） | 需客户签字 | 验收报告原稿、服务明细单原稿 |
| | 验收签字件（已签） | 已完成签字 | 已签验收单、已签服务明细单 |
| | 验收报告 | 验收结论 | 验收总结、验收标准、验收测试报告 |
| | 项目总结 | 复盘归档 | 项目总结、经验教训、改进计划 |
| **转客户成功** | 交接文档 | 移交材料 | 交接清单、系统账号、权限说明 |
| | 培训资料 | 培训相关 | 操作手册、培训PPT、视频教程 |
| | FAQ | 常见问题 | 问题集、解决方案、故障排除 |
| **关闭** | 项目归档 | 归档材料 | 归档清单、最终报告、财务结算 |
| | 复盘总结 | 经验沉淀 | 复盘纪要、改进计划、最佳实践 |
| **未分类** | — | 尚未被AI分析 | 新上传的文件默认状态 |

**设计原则：**
- 文件分类阶段保持**细粒度**（10个阶段 + 子分类），因为：
  1. 文件内容需要精确分类以便检索
  2. 不同阶段的文件有不同处理逻辑
  3. AI需要更精细的判断来准确分类
- **子分类按文档用途分**，不按文件格式分（pdf/doc不作为分类依据）
- **验收阶段区分文件状态**：待签 vs 已签，同一类文件区分处理阶段
- **会议纪要保留在各阶段内**，不单独做跨阶段类别，AI通过会议内容判断阶段归属

### 1.2 两套阶段的关系

```
项目阶段（3个）          文件分类阶段（11个）
┌─────────────┐         ┌─────────────────────┐
│   售前      │ ←─────→ │ 售前、启动           │
├─────────────┤         ├─────────────────────┤
│   进行中    │ ←─────→ │ 需求、方案、构建、    │
│             │         │ 测试、上线、验收、    │
│             │         │ 转客户成功           │
├─────────────┤         ├─────────────────────┤
│   关闭      │ ←─────→ │ 关闭                 │
└─────────────┘         └─────────────────────┘
```

**关键区别：**
- 项目阶段是**项目级**属性，一个项目只有一个当前阶段
- 文件分类阶段是**文件级**属性，每个文件有一个分类阶段
- AI分类文件时，会同时判断文件的分类阶段，并可能触发项目阶段推进

### 1.3 阶段推进规则

当AI分类文件时，如果识别到文件属于更高阶段，会弹窗询问是否推进项目阶段：

| 当前项目阶段 | 文件分类阶段 | 是否触发推进询问 |
|--------------|--------------|------------------|
| 售前 | 售前、启动 | 否 |
| 售前 | 需求、方案、构建、测试、上线、验收、转客户成功、关闭 | **是** → 询问是否推进到「进行中」 |
| 进行中 | 需求、方案、构建、测试、上线、验收、转客户成功 | 否 |
| 进行中 | 关闭 | **是** → 询问是否推进到「关闭」 |
| 关闭 | 任何阶段 | 否（已归档） |

## 二、产品定位与目标用户

### 2.1 产品定位

**项目管理助手**是一款面向项目管理人员的 Windows 桌面端 AI 驱动工具。核心价值：

1. **集中管理**：将散落在邮件、U盘、微信中的项目文件集中到统一位置
2. **智能分类**：AI 自动识别文件内容，归类到对应的文件分类阶段（11个）
3. **阶段感知**：通过文件内容自动判断项目整体进展（3个项目阶段）
4. **AI 助手**：基于项目全部文件上下文，提供分析、问答、摘要能力

### 2.2 目标用户与角色

| 角色 | 典型场景 | 关注重点 |
|------|---------|---------|
| **项目经理（PM）** | 管理多个客户项目的文件、跟进进度、协调团队 | 项目全局视图、进度里程碑、风险识别 |
| **售前解决方案** | 管理投标文件、技术方案、客户沟通记录 | 报价单管理、方案文档分类、客户信息提取 |
| **开发工程师** | 管理技术方案、代码规范、接口文档、测试报告 | 技术文档分类、代码规范管理 |
| **客户成功经理** | 管理验收文档、培训资料、交接清单 | 验收进度、签字状态、交接完整性 |

## 三、功能需求详述

### F01 项目管理

#### F01.1 创建项目

**业务场景**：用户开始管理一个新客户项目。

**需求描述**：
1. 用户输入项目名称（必填）
2. 选择文件分类方式：
   - **按阶段分类**：预定义 11 个文件分类阶段，文件自动归类到对应阶段文件夹
   - **按内容分类**：AI 自动识别内容，动态创建分类文件夹
3. 项目创建后：
   - 在用户指定的存储位置（默认 `{userData}/projects`，可自定义）创建项目文件夹
   - 自动创建 11 个阶段子文件夹（按阶段分类时）
   - 自动创建 `.ai/` 元数据目录
   - **项目阶段默认为「售前」**
4. 文件系统创建失败时回滚数据库记录

**验收标准**：
- [ ] 输入名称后能成功创建项目
- [ ] 文件夹在正确位置创建
- [ ] 阶段子文件夹完整（11个）
- [ ] 项目阶段默认为「售前」
- [ ] 创建失败时数据库不残留孤儿记录

#### F01.2 项目列表

**业务场景**：用户管理多个项目，需要快速找到目标项目。

**需求描述**：
1. 支持**卡片视图**和**表格视图**切换
2. 卡片视图：
   - 每张卡片包含：顶部彩色条（4px，颜色跟随项目状态）、项目名称、状态标签、文件数量、分类方式标签、当前阶段、更新时间
   - 网格布局：`repeat(auto-fill, minmax(320px, 1fr))`，gap 20px
   - Hover 效果：阴影变化 + 微上移
   - Hover 时显示编辑/删除按钮
3. 表格视图：标准 AntD Table
4. 搜索：按项目名称模糊搜索
5. 筛选：按**项目阶段**（售前/进行中/关闭）
6. 排序：按更新时间、创建时间、名称

**验收标准**：
- [ ] 卡片视图布局正确，无遮挡
- [ ] 搜索实时过滤
- [ ] 项目阶段筛选有效
- [ ] 排序正确

#### F01.3 编辑项目

**业务场景**：用户需要修改项目信息。

**需求描述**：
1. 支持修改**项目阶段**（售前/进行中/关闭）
2. 支持修改项目名称
3. 支持修改分类方式（从阶段分类改为内容分类，或反之）
4. 修改后立即在界面反映

#### F01.4 删除项目

**业务场景**：用户需要删除不再管理的项目。

**需求描述**：
1. 二次确认弹窗
2. 级联删除：数据库记录 + 物理文件夹 + 对话记录
3. 删除后列表自动刷新

#### F01.5 打开文件夹

**业务场景**：用户需要在系统文件管理器中查看项目文件。

**需求描述**：一键打开项目在本地磁盘上的文件夹。

### F02 文件管理

#### F02.1 上传文件

**业务场景**：用户将项目文件导入系统。

**需求描述**：
1. 支持拖拽上传和点击上传
2. 支持的文件类型：TXT、MD、CSV、DOCX、XLSX、PDF、PNG、JPG
3. 单文件大小限制：50MB（前端 + 后端双重校验）
4. 上传后文件显示在文件列表中，**分类阶段默认为「未分类」**
5. 上传后自动提取文件内容（用于 AI 分析）

**验收标准**：
- [ ] 拖拽和点击上传均可用
- [ ] 超过 50MB 的文件被拒绝
- [ ] 上传后内容自动提取
- [ ] 新文件分类阶段为「未分类」

#### F02.2 文件列表

**业务场景**：用户查看和管理项目文件。

**需求描述**：
1. 显示：文件名、文件类型、文件大小、**文件分类阶段**、是否已分析、是否含签字
2. 左侧边栏按**文件分类阶段**分组筛选（显示每个阶段的文件数量）
3. 支持复选框多选
4. 每个文件旁有操作按钮：AI 分类、打开文件、删除

#### F02.3 删除文件

**需求描述**：
1. 二次确认
2. 同时删除物理文件和数据库记录
3. 路径安全校验（防止路径遍历）

#### F02.4 打开文件

**需求描述**：调用系统默认程序打开文件。

#### F02.5 手动分类

**业务场景**：用户发现文件分类错误，手动修正。

**需求描述**：
1. 每个文件旁提供下拉菜单，可选择 11 个**文件分类阶段**
2. 手动分类后文件物理移动到对应阶段文件夹
3. 数据库记录同步更新

### F03 AI 智能分类

#### F03.1 单文件分类

**业务场景**：用户对单个文件触发 AI 分析。

**需求描述**：
1. 点击文件旁的"分类"按钮
2. AI 读取文件内容，判断：
   - **文件分类阶段**（11个阶段之一）
   - **项目阶段**（售前/进行中/关闭）— 用于判断是否需要推进
   - 文件内容摘要
   - 关键信息（项目编号、合同号等）
3. 分类完成后更新数据库 + 移动物理文件
4. 自动检测签字（PDF/图片）
5. **如果文件阶段高于当前项目阶段，弹窗询问是否推进项目阶段**

#### F03.2 一键分类

**业务场景**：用户有大量未分类文件，需要批量处理。

**需求描述**：
1. 点击"一键分类"按钮，对所有**分类阶段为「未分类」**的文件批量触发 AI 分类
2. 显示进度（当前/总数）
3. 支持取消
4. 分类完成后自动刷新

#### F03.3 分类 Prompt

**需求描述**：
1. 分类 Prompt 支持用户自定义（设置页面）
2. 默认 Prompt 包含 10 个**文件分类阶段**及各阶段**子分类**的描述
3. AI分类时同时判断：(1)文件属于哪个阶段 (2)属于该阶段的哪个子分类
4. 验收阶段额外判断文件状态：待签 vs 已签
5. 会议纪要类文件通过内容关键词判断归属阶段
6. 保存后立即生效

### F04 阶段管理

#### F04.1 项目阶段定义

**业务场景**：项目整体状态管理。

**需求描述**：
1. **项目阶段固定为3个**：售前、进行中、关闭
2. 每个阶段有固定的颜色和图标
3. 项目阶段不可自定义（保持简单）

#### F04.2 文件分类阶段定义

**业务场景**：文件精细化分类，含子分类。

**需求描述**：
1. **文件分类阶段默认10个 + 未分类**：售前、启动、需求、方案、构建、测试、上线、验收、转客户成功、关闭
2. 每个阶段下有**子分类**（3-7个），按文档用途区分（见§1.1子分类表）
3. 每个阶段有固定的颜色和图标（见设计规范）
4. 用户可在设置中自定义文件分类阶段（新增、删除、排序）
5. 子分类支持用户自定义（新增、删除）
6. 自定义阶段和子分类应用于新创建的项目
7. 已有项目不受影响

#### F04.3 自动阶段推进

**业务场景**：AI 识别到文件属于更高阶段时，提醒用户推进**项目阶段**。

**需求描述**：
1. AI 分类文件时，同时判断文件的**文件分类阶段**
2. 如果文件分类阶段高于当前**项目阶段**，弹窗询问是否推进
3. 示例：项目在「售前」，AI 识别到合同文件分类为「需求」→ 弹窗询问是否推进到「进行中」
4. 用户确认后自动更新**项目阶段**

#### F04.4 手动阶段推进

**需求描述**：用户可手动将**项目阶段**推进到下一阶段，需二次确认。

#### F04.5 阶段展示

**需求描述**：
1. 左侧边栏按**文件分类阶段**分组显示文件，每个阶段旁显示文件数量

## 四、非功能性需求

| 编号 | 需求 | 描述 |

| N01 | 数据安全 | API Key 使用 safeStorage 加密存储，界面显示掩码 |

## 五、术语表

| 术语 | 含义 |
|------|------|
| 项目阶段 | 项目整体状态，固定3个：售前、进行中、关闭 |
| 文件分类阶段 | 文件精细化分类，默认11个：售前、启动、需求、方案、构建、测试、上线、验收、转客户成功、关闭、未分类 |
| 阶段推进 | 当AI识别到文件属于更高阶段时，询问用户是否推进项目阶段 |
| 分类方式 | 项目创建时选择的文件组织方式：按阶段分类或按内容分类 |
