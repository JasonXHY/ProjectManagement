# PMAer v0.2.0 需求文档

> 版本：1.0 | 日期：2026-06-22 | 状态：正式发布
> 前置文档：v0.2.0-plan-final.md（规划终稿）

---

## 一、版本概述

### 1.1 版本目标

v0.2.0 是 PMAer 的**体验深化版本**，围绕三个核心目标：

1. **详情页卡片系统** — 按项目阶段差异化展示关键信息
2. **项目转交功能** — 支持整体打包和AI辅助选择性转交
3. **Markdown预览** — 统一渲染能力，提升文档查看体验

### 1.2 不包含的功能

以下功能推迟至后续版本：

| 功能 | 推迟原因 |
|------|----------|
| 「更多卡片」自定义 | 卡片数量少时过度设计 |
| 暗色模式 | 需要全量样式重构 |
| 快捷键 | 锦上添花 |
| 拖拽排序 | 锦上添花 |
| 虚拟滚动 | 内测阶段文件量不大 |
| 搜索增强 | 需要先积累足够项目数据 |
| 数据导出 | 独立功能，可单独做版本 |
| 项目归档 | 关闭阶段够用 |
| 多语言 | 内测用户为中文用户 |

## 二、功能需求

### FR-001：详情页统计行

**优先级**：P0

**描述**：详情页顶部显示4个统计卡片，提供项目关键指标概览。

**功能要求**：

| 编号 | 卡片 | 内容 | 交互 |
|------|------|------|------|
| FR-001-1 | 文件数量 | 项目文件总数 | 无 |
| FR-001-2 | 里程碑 | 下一步里程碑名称 + 日期 | 点击弹窗查看完整时间轴 |
| FR-001-3 | 待处理 | 未被AI分析的文件数 | 无 |
| FR-001-4 | AI摘要 | 「查看摘要」「生成/更新」按钮 | 查看摘要打开800px Markdown渲染弹窗 |

**验收标准**：
- [ ] 4个卡片等宽网格显示
- [ ] 文件数量从数据库实时计算
- [ ] 里程碑卡片显示下一步节点名称和日期
- [ ] 无里程碑数据时显示"—"
- [ ] 待处理数量正确计算
- [ ] AI摘要弹窗使用Markdown渲染

## 六、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AI文件推荐不准确 | 协作转交体验差 | 手动调整入口 + 推荐理由 |
| 转交包体积过大 | 传输困难 | 协作转交精简包 + 100MB提示 |
| Markdown XSS | 安全风险 | rehype-sanitize过滤 |
| 跨版本兼容 | 旧版无法导入 | format_version校验 + 向后兼容 |
| 同名项目冲突 | 数据覆盖 | 提示重命名 + 默认不覆盖 |
