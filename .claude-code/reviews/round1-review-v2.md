## 1. 三层样式架构评估

### 1.1 架构可行性：✅ 完全可行

三层分工明确，职责边界清晰：

| 层级 | 职责 | 工具 | 边界 |
|------|------|------|------|
| 第一层 | AntD 组件主题 Token | ConfigProvider | 组件内部的颜色、圆角、字号等 |
| 第二层 | 布局相关 | Tailwind（禁用 preflight） | flex、grid、padding、margin、width、height |
| 第三层 | 细节覆盖 | overrides.css | ConfigProvider 无法覆盖的样式 |

### 1.2 边界模糊风险预判

**可能的边界模糊场景：**

1. **按钮 hover 颜色**：属于"颜色"（第一层）还是"交互反馈"（第三层）？
   - **建议**：优先在 ConfigProvider 中通过 `colorPrimaryHover` 等 Token 覆盖，这是正确的做法。

2. **卡片阴影**：属于"阴影"（第一层）还是"视觉效果"（第三层）？
   - **建议**：阴影 Token 已在 ConfigProvider 中定义，优先使用 Token。

3. **Menu 选中指示条**：属于"选中状态"（第一层）还是"装饰元素"（第三层）？
   - **建议**：ConfigProvider 的 `itemSelectedBg` 可以实现背景色，左侧 3px 指示条需要 CSS 覆盖（第三层），这是合理的划分。

### 1.3 总体评价

三层架构方案务实可行，避免了 Tailwind 和 AntD 在同一元素上"打架"。关键是严格遵守"布局用 Tailwind，组件用 AntD Token，细节用 CSS 覆盖"的原则。

## 2. 组件简化方案评估

### 2.1 视图切换：Radio.Group buttonStyle="solid"

**评估：✅ 合理**

- AntD 6 的 `Radio.Group` + `buttonStyle="solid"` 可以实现紧凑的按钮组效果
- 比自定义 toggle 更稳定，自动处理受控状态
- **视觉匹配度**：与 mockup 中的 `.view-toggle` 效果接近，可以通过 ConfigProvider 的 `Radio` Token 调整颜色和圆角

### 2.2 上下文面板：Drawer 替代常驻侧边栏

**评估：✅ 合理，且优于原方案**

优点：
- 不需要处理面板折叠/展开的布局动画
- 关闭时对话区自动占满宽度，无需手动调整 flex 布局
- 移动端/小窗口下自动表现良好
- **视觉匹配度**：Drawer 弹出效果与 mockup 的交互逻辑一致，只是触发方式不同（点击按钮弹出 vs 常驻切换）

**建议**：Drawer 的 `width` 设置为 240px（与 mockup 一致），`mask` 设置为 `false`（不显示遮罩层），这样更接近常驻面板的体验。

### 2.3 文件上传：Upload.Dragger 替代自定义拖拽区

**评估：✅ 合理**

- AntD 6 的 `Upload.Dragger` 组件已经实现了完整的拖拽交互逻辑
- 通过 ConfigProvider 和 CSS 覆盖可以自定义其内部样式
- **视觉匹配度**：mockup 中的拖拽区样式（虚线边框、图标、文字、格式标签）可以通过 CSS 覆盖实现

**建议**：Upload.Dragger 的默认样式与 mockup 差异较大，需要在 `overrides.css` 中进行较多的样式覆盖。建议在 Task 1.5 样式验证页中重点测试。

### 2.4 StageNav：Menu 替代自定义列表

**评估：✅ 合理**

- AntD 6 的 `Menu` 组件 `mode="inline"` 模式非常适合侧边栏导航
- 通过 ConfigProvider 的 `Menu` Token 可以定制选中态、hover 态等
- `inlineCollapsed` 属性天然支持折叠功能
- **视觉匹配度**：mockup 中的导航项样式（图标、文字、badge、选中指示条）可以通过 Menu 的 `items` render 和 CSS 覆盖实现

**建议**：Menu 的默认样式与 mockup 差异主要在选中指示条（左侧 3px 条），需要在 `overrides.css` 中用 `::before` 伪元素实现。

### 2.5 消息气泡：纯自定义 div

**评估：✅ 合理，且是最佳方案**

- 消息气泡的样式（颜色、圆角、对齐方式）完全自定义，不依赖任何 AntD 组件
- CSS 控制气泡形状和颜色，灵活性最高
- **视觉匹配度**：mockup 已经给出了完整的样式定义，纯 div + CSS 可以精确还原

### 2.6 设置页：Tabs + 配置区块

**评估：✅ 合理**

- AntD 6 的 `Tabs` 组件已经是最稳定的选择
- 配置区块用 div + 边框 + 圆角实现，不需要封装通用 Card 组件
- **视觉匹配度**：mockup 中的设置页样式可以通过 Tabs + CSS 覆盖实现

### 2.7 总体评价

所有简化方案都合理且可行。关键点：
- 能用 AntD 组件的优先用 AntD 组件（稳定性高）
- 样式差异大的用 CSS 覆盖（灵活性高）
- 完全自定义的用纯 div + CSS（控制力强）
