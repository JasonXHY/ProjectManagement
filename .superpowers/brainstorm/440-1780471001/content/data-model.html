数据模型设计

SQLite数据库表结构设计

📁 项目表 (projects)
id INTEGER PRIMARY KEY
name TEXT NOT NULL
description TEXT
created_at DATETIME
updated_at DATETIME
status TEXT DEFAULT 'active'
🔗 一个项目包含多个任务
🔗 一个项目包含多个文件
🔗 一个项目包含多条对话记录
✅ 任务表 (tasks)
id INTEGER PRIMARY KEY
project_id INTEGER REFERENCES projects(id)
name TEXT NOT NULL
start_date DATE
end_date DATE
progress INTEGER DEFAULT 0
status TEXT DEFAULT 'pending'
🔗 任务属于一个项目
🔗 任务可以依赖其他任务
📄 文件表 (files)
id INTEGER PRIMARY KEY
project_id INTEGER REFERENCES projects(id)
name TEXT NOT NULL
path TEXT NOT NULL
category TEXT
tags TEXT
version INTEGER DEFAULT 1
content_hash TEXT
🔗 文件属于一个项目
🔗 文件有版本历史
💬 对话表 (conversations)
id INTEGER PRIMARY KEY
project_id INTEGER REFERENCES projects(id)
role TEXT NOT NULL
content TEXT NOT NULL
created_at DATETIME
token_count INTEGER
🔗 对话属于一个项目
🔗 记录用户和AI的交互
💡 数据模型设计要点

• 简洁明了：只包含必要的字段，避免过度设计

• 外键关联：通过project_id建立表间关系

• 版本控制：文件表包含version和content_hash字段

• Token追踪：对话表记录token使用量，便于成本控制

🔧 扩展性考虑

• 后期可添加：里程碑表(milestones)、依赖关系表(dependencies)

• 文件标签系统：可将tags字段改为独立的标签表

• 向量索引：可单独存储，不放在主数据库中
